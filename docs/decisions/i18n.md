# Decisions — Internationalisation (i18n: pt + en)

_Added when English support landed alongside Portuguese. Phased rollout:
infra + nav/auth/settings first (this file), then one module per phase._

## Stack & setup

- **`i18next` + `react-i18next` + `i18next-browser-languagedetector`.** Chosen
  over a custom context for per-language plural rules, interpolation, namespaces,
  and lazy-load headroom. Init lives in `frontend/src/i18n/index.ts`, imported at
  the top of `main.tsx` **before render** (formatters read the i18n singleton).
- **Namespaces per module:** `common, nav, auth, settings` (Phase 1) + later
  `overview, portfolio, budget, loan, compare`. Files:
  `frontend/src/i18n/locales/<lng>/<ns>.json`. Keep `pt/` and `en/` in **key
  parity** (a node diff script in the verify step catches gaps; dev logs missing
  keys via `missingKeyHandler`).
- **Type-safe keys:** `frontend/src/i18n/i18next.d.ts` augments i18next with the
  `pt` resources as the canonical key shape, so a missing/typo'd `t('…')` key is
  a **compile error**. Consequence: dynamic keys must be typed as literal unions,
  not `string` (see `SECTION_KEYS` in `Layout.tsx` and `as const` on
  `GESTAO_ITEMS` in `BottomNav.tsx`) — a bare `string` won't satisfy `t()`.

## Decisions

- **Default = auto-detect, fallback pt.** Detector order `['localStorage',
  'navigator']`, cached to `localStorage` key `wallet360.lang`, `fallbackLng:
  'pt'`, `load: 'languageOnly'` (so `en-US`→`en`, `pt-PT`→`pt`). English browsers
  see English; everyone else Portuguese.

- **Persistence = localStorage + DB.** localStorage (via the detector) is the
  immediate source of truth and works **before login / on auth pages**. For
  signed-in users we also persist to `PortfolioSettings.language` and hydrate
  from it on login (`useLanguageSync` in `Layout`), so the choice follows them
  across devices. **DB wins once authenticated.** Picker: `LanguageSection.tsx`
  → `useChangeLanguage` (`i18n.changeLanguage` + `useUpdateSettings({language})`,
  same pattern as the watchlist). DB sync failure is non-fatal (local already
  applied).
  - Schema: nullable `language` on `PortfolioSettings` in **both** schemas +
    migration `add_settings_language` + `export.ts`/`import.ts` round-trip.
    **Prod `db push` is deferred to the owner** (Neon snapshot first; nullable
    column ⇒ additive/non-destructive).

- **Remount-on-language-change.** `App.tsx` keys the routed tree with
  `key={i18n.language}`. Providers (router/query/auth) stay mounted; only the
  current page remounts. Guarantees number/date formats and any non-`t` subtree
  refresh instantly, and avoids half-translated UI during the phased rollout.

- **Locale-aware formatting, EUR always.** `format.ts` maps the active language →
  Intl locale (`en`→`en-IE` "€1,234.56", `pt`→`pt-PT` "1.234,56 €"), with
  per-locale memoized formatters and per-locale month arrays. Currency stays EUR
  in every language. `format.ts` reads the i18n singleton directly (pure, no
  React) — re-renders come from the remount + `useTranslation`.

- **Categories: translate the LABEL, not the stored value.** `Income.category` /
  `Expense.category` are persisted as their **canonical Portuguese string**
  (e.g. `Habitação`). `categoryLabel(canonical)` (in `categoryDictionary.ts`)
  maps that canonical string → `common:categories.*` for display only. Selects
  submit the canonical pt value unchanged. **Regression to watch:** an expense
  added in English mode must still store `Habitação`, not `Housing`. The
  keyword→category parsing `DICTIONARY` stays Portuguese (it parses PT bank
  statements, not UI).

## Out of scope / deferred

- **Backend-originated messages** (API validation/auth errors returned as pt
  text, plus the `api.ts` "Erro {status}" fallback) are still Portuguese.
  Translating them needs the API to return codes the frontend maps — a later pass.
- **Google Sign-In button** localizes itself via Google's own widget, not our
  i18n (so it can show pt while the page is en). Pass a locale to the GSI button
  if this matters.
- **Phases 2–5:** Overview, Portfolio, Budget, Loan + Compare not yet converted.
