ENVIRONMENT VARIABLES  (Vercel → Settings → Environment Variables)
--------------------------------------------------------------------------------

DATABASE_URL          Neon pooled connection string
                      Host must end in  -pooler.REGION.aws.neon.tech
                      Query string:  ?sslmode=require&channel_binding=require

SESSION_SECRET        Random string, ≥ 32 characters

GOOGLE_CLIENT_ID      From Google Cloud Console (OAuth 2.0 client)
VITE_GOOGLE_CLIENT_ID Same value as GOOGLE_CLIENT_ID (exposed to frontend)

APP_ORIGIN            Full URL of the deployed app, e.g. https://wallet360.vercel.app
                      Used for CORS and cookie domain

GOCARDLESS_SECRET_ID  (see GoCardless section below — NOT YET configured)
GOCARDLESS_SECRET_KEY (see GoCardless section below — NOT YET configured)

FINNHUB_API_KEY       Optional — used for quotes fallback. Get free key at
                      https://finnhub.io  (Yahoo Finance is primary, no key needed)

ALLOWED_ORIGINS       Leave empty if frontend and API are on same Vercel domain

GoCardless was the only piece not completed. New signups at
bankaccountdata.gocardless.com were temporarily disabled.

Steps to complete when signups re-open:
  1. Go to https://bankaccountdata.gocardless.com
  2. Create a new account / log in
  3. Navigate to  User secrets  →  Create new secret
     Name it: wallet360-production
  4. Copy the Secret ID and Secret Key
  5. In Vercel → Settings → Environment Variables, add:
       GOCARDLESS_SECRET_ID = <your secret id>
       GOCARDLESS_SECRET_KEY = <your secret key>
  6. Trigger a redeploy (Deployments → Redeploy latest)

KNOWN ISSUES / OPTIONAL IMPROVEMENTS (not blocking)
--------------------------------------------------------------------------------

1. BUNDLE SIZE WARNING
   Vite build warns about chunks > 500 KB:
     index-*.js        ~581 KB raw  (178 KB gzip)
     pdfStatementParser-*.js  ~367 KB raw  (109 KB gzip)
   The PDF parser is only needed when the user picks a PDF file. It should
   be lazy-loaded:
     // In the import file
     const parsePdf = lazy(() => import('./pdfStatementParser'))
   This would cut the initial bundle by ~30%.

2. FX CONVERSION FOR TICKER PRICES
   When a user adds a portfolio asset via ticker search, Yahoo returns the
   price in the asset's native currency (USD for US stocks, GBX for some
   London ETFs, etc.).  The current auto-fill uses the raw price without
   converting to EUR.
   The backend already has convertPrice() in backend/src/lib/fx.ts —
   it can be wired up via a new route or in the quotes/search response.
   Until then, users should manually correct the EUR value if needed.

3. YAHOO FINANCE UNOFFICIAL API
   The ticker search uses query1.finance.yahoo.com/v1/finance/search —
   this is undocumented (same endpoint as yahoo.com's search bar).
   Works well in practice. If it starts failing, Finnhub's
   GET /api/v1/search?q=<query>&token=<key> is the backup.

RUNNING LOCALLY
--------------------------------------------------------------------------------

# Install deps
npm install

# Dev mode (frontend + backend in parallel)
npm run dev

# Type-check only
npm run typecheck       # runs tsc --noEmit in both workspaces

# Build for production
npm run build           # outputs to dist/

# Database (dev)
cd backend && npm run db:generate   # generate Prisma client
cd backend && npm run db:push       # push schema to dev SQLite

# Database (prod — run manually or triggered by Vercel build)
cd backend && npm run db:push:prod  # push schema to Neon
